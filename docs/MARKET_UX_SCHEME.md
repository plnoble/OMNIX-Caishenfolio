# 行情 UI / K 线 / 缓存 方案（调研结论）

面向 Caishenfolio 本地研究台。参考：东方财富 / 同花顺 / 雪球 / TradingView / 富途牛牛 的常见交互。

---

## 1. 主流行情软件 K 线怎么做

### 1.1 周期（几乎标配）

| 层级 | 常见项 | 说明 |
| --- | --- | --- |
| 分时 | 分时、五日 | 实时盘口，研究台可二期 |
| 短周期 | 1/5/15/30/60 分钟 | 做 T、短线；需分钟线源 |
| **日线族** | **日K、周K、月K** | **研究/中长线默认** |
| 长周期 | 季K、年K | 有的软件有，使用频率低于日月周 |
| 自定义 | N 日 / 自定义区间 | TradingView 强；国内软件多为固定枚举 |

**结论（我们现阶段）：**

1. **默认日K**（打开即看最近交易日附近）。
2. 一键：**日K / 周K / 月K**（已具备 API + UI 骨架）。
3. **年K**：可作为「月K 聚合」或单独周期，优先级低于日月周。
4. 分时/分钟线：等稳定实时源 + 明确需求后再做（数据量与权限都更大）。

### 1.2 图表交互（标配体验）

| 能力 | 主流做法 | 我们建议阶段 |
| --- | --- | --- |
| 蜡烛图红涨绿跌（A股习惯） | 标配 | **P3 已有简易版** |
| 默认看「最近一段」 | 日K 约 3～12 个月；周K 约 2～5 年；月K 约 5～10 年 | **P3 立刻做默认区间** |
| 缩放/拖拽平移 | 鼠标滚轮 + 拖拽 | P3.1 简易 / P4 完整 |
| MA5/10/20/60 | 均线叠层 | P3.1 |
| 成交量副图 | 主图下方量柱 | P3.1 |
| 十字光标 + 悬浮 OHLC | 悬停显示 | P3.1 |
| 复权：前复权/后复权/不复权 | 工具栏切换 | P3.1（数据源已有 adjustment） |
| 全屏 / 多图对比 | 高级 | 后期 |

### 1.3 推荐默认打开方式（对标「打开软件先看最新」）

```
默认周期 = 日K
结束日   = 最近一个交易日（休市则回退到上一交易日）
开始日   = 结束日往前约 120～250 个自然日（约半年～一年日K）
用户可改开始/结束；切换周期时，若用户未「锁定自定义区间」，则按周期重置推荐区间
```

| 周期 | 推荐默认跨度 | 理由 |
| --- | --- | --- |
| 日K | 约 6～12 个月 | 趋势 + 波段够用，点不太密 |
| 周K | 约 3～5 年 | 中线结构 |
| 月K | 约 8～15 年 | 长线位置 |

## 2. 开始/结束时间：必须默认「最新」

**问题：** 写死 `2024-01-02 ~ 2024-01-12` 不符合「先看最新」习惯。

**方案：**

1. 启动时自动填：**结束 = 今天（或上一个工作日）**，**开始 = 按周期推算**。
2. 提供快捷按钮（建议下一迭代）：`近1月` `近3月` `近1年` `近3年` `今年以来` `全部`。
3. 手动改日期永远优先；点「重置到最新」一键回到默认。
4. 加载时若结束日晚于最后有数据日，以数据源实际最后一根为准（不报错、不造数）。

## 3. 关注列表：是否本地缓存全部历史？

### 3.1 你的直觉是对的

| 做法 | 结果 |
| --- | --- |
| 每次打开都全量拉网 | 慢、费流量、上游限流 |
| 关注股「全历史一次拉完 + 增量更新」 | 主流软件 / 本地库标准做法 |
| 全放内存 | **不推荐**（标的一多会炸） |

### 3.2 推荐架构：**磁盘冷热分层，内存只放当前屏**

```
┌─────────────────────────────────────────┐
│  关注列表 (watchlist.json)               │
└─────────────────┬───────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────┐
│  本地 Bars 库（State 或独立 Market 根）    │
│  建议：SQLite 或 Parquet/按标的分文件      │
│  键：symbol + interval + trade_date       │
│  字段：OHLCV + provider + 复权 + 写入时间  │
└─────────────────┬───────────────────────┘
                  │ 启动 / 定时 / 手动刷新
                  ▼
┌─────────────────────────────────────────┐
│  增量同步任务                             │
│  1) 读本地 max(trade_date)               │
│  2) 只请求 (max_date, 今天] 的缺口         │
│  3) 合并写入；失败 fail-closed 不写假数    │
└─────────────────┬───────────────────────┘
                  │ 用户点开某只关注
                  ▼
┌─────────────────────────────────────────┐
│  内存：仅当前标的 + 当前周期的可视窗口     │
│  （例如日K 最多加载 2～5 年到图上）         │
└─────────────────────────────────────────┘
```

### 3.3 容量粗算（日K，单标的）

| 假设 | 数量级 |
| --- | --- |
| A 股一年约 240 根日K | ~240 行 |
| 20 年全日K | ~5,000 行 |
| 一行 OHLCV ~100～200 字节（含索引放大） | 单标的 20 年 **&lt; 2～5 MB** 级 |
| 关注 50 只 × 20 年日K | 大约 **百 MB 磁盘内**，完全可接受 |
| 若再存分钟线 1 年 | 会大 1～2 个数量级 → **默认不缓存分钟线** |

**内存：** 同时只渲染 1 只（或对比 2～4 只）× 当前窗口几千根 → **几十 MB 内**，与「全市场进内存」无关。

### 3.4 同步策略（打开软件每天）

| 时机 | 动作 |
| --- | --- |
| 启动且核心就绪 | 后台任务：对关注列表逐个 **增量补日K**（优先日K） |
| 交易日 15:30 后 / 次日首次打开 | 拉「上一交易日」缺口 |
| 用户点某只 | 先读本地秒开图 → 后台静默增量 → 有新数据再刷新 |
| 周K/月K | 可由日K本地聚合 **或** 单独缓存；优先本地聚合省请求 |
| 失败 | 保留旧缓存 + 角标「数据可能不是最新」；绝不填假 K |

### 3.5 建议落地阶段

| 阶段 | 内容 |
| --- | --- |
| **P3 现在** | 默认最新日期区间；关注列表已有；仍实时拉取 |
| **P3.5** | 本地 SQLite/Parquet bars 缓存 + 增量 API |
| **P4** | 启动后台同步、进度条、缓存清理、按磁盘配额 LRU |

**清理策略：** 未关注且 90 天未打开的标的可删缓存；用户可「清理行情缓存」。

## 4. 有了「输入 + 加入关注」，搜索还有没有用？

**有用，而且应该换定位，不是删掉。**

| 入口 | 解决什么问题 | 谁用 |
| --- | --- | --- |
| **关注列表** | 日常看的 10～50 只，一键打开 | 每天 |
| **直接输入代码** | 你已经知道 `600000` / `SSE:600000` | 熟手 |
| **搜索** | **不知道代码、只记得名字**（「宁德」「茅台」「腾讯」） | 新手 / 偶发 |
| **浏览/热榜**（后期） | 发现新标的 | 研究 |

### 4.1 建议产品结构（左侧）

```
┌ 我的关注 ─────────────┐
│ 双击打开 / 右键删      │  ← 主路径
└────────────────────────┘
┌ 添加标的 ─────────────┐
│ [搜索框] 名称或代码    │  ← 搜索降级为「添加」
│ 结果列表 → 加入关注    │
│ 或：直接打开预览       │
└────────────────────────┘
```

- 搜索**不是**每天主路径，是 **「找票 → 加入关注」的发现工具**。
- 主路径应是：**关注列表 → 自动加载最新日K（优先本地缓存）**。
- 输入框保留：专业用户快捷；可支持只输 `600000` 自动补全交易所。

### 4.2 交互细节建议

1. 搜索结果项：`[A股] 浦发银行` + 次要文字 `SSE:600000`。
2. 按钮：**打开** | **加入关注**（不必先填中间框再点关注）。
3. 关注项双击：打开 = 填标的 + 默认最新区间 + 日K + 自动加载。
4. 若搜索长期慢：本地先搜「关注+历史打开过的标的」，再请求在线搜索。

## 5. 总体路线图（建议）

| 优先级 | 项 | 状态 |
| --- | --- | --- |
| P0 | 默认最新交易区间 + 日K | **应立即做** |
| P0 | 关注双击自动加载最新 | 部分有，补齐默认区间 |
| P1 | 日/周/月切换 + 均线/成交量副图 | 周期已有，图表增强 |
| P1 | 搜索改为「添加标的」文案与一键关注 | 文案/按钮 |
| P2 | 本地 bars 缓存 + 增量更新 | 磁盘不占内存 |
| P3 | 分时/分钟、年K、多股对比 | 按需 |

---

## 6. 一句话结论

1. **K 线：** 默认日K + 最近半年～一年；周/月为切换；图表补均线/量/十字光标。  
2. **时间：** 结束日=最近交易日，开始日按周期推算，可手改。  
3. **缓存：** 关注股历史放**磁盘**，内存只放当前图；每日增量补最新，不占爆内存。  
4. **搜索：** 不删，定位为「发现并加入关注」；日常主路径是关注列表。
